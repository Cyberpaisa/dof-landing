05 · Lecciones aprendidas

L-XX — nuevas reglas.

L-2026-04-16-01 · CRÍTICA
Evolution tests destruyen git state
test_evolution_* crea branches evolution-checkpoint-<ts>, hace commit, y git checkout main — lo cual descarta todos los cambios no commiteados en el working tree. Mitigación: commitear ANTES de ejecutar o usar SKIP_EVOLUTION_CHECKPOINT=1.
L-2026-04-16-02 · IMPORTANTE
Team agents en paralelo acelera auditoría
3 Auditores (Explore subagents) con scope acotado y formato de salida obligatorio entregaron diagnóstico completo en <3 min. Cada uno con ≤500 palabras. Mejor ratio señal/ruido que auditor único.
L-2026-04-16-03 · DOCUMENTACIÓN
Docs canónicos drift silenciosamente
CLAUDE.md tenía 3 secciones contradictorias (v0.8.0 / v0.5.1 / Sesión 5) porque cada sesión agregó sin consolidar. Regla: en cada sesión, unificar estado actual en una sola sección + mover histórico a CHANGELOG.md.
L-2026-04-16-04 · DEPS
requirements.txt vs pyproject.toml drift
requirements.txt pinea crewai==0.11.2 (2024) mientras pyproject.toml declara >=0.108.0. Pin viejo instalaba versión con API diferente (BaseTool cambió). Alinear a pyproject (autoritativo).
L-2026-04-16-05 · GIT
Archivos dentro de dir gitignoreado no se re-incluyen con !
docs/09_sessions/ excluye todo. !docs/09_sessions/CHANGELOG.md NO funciona. Solución: git add -f path para force-add específicos, o patrón docs/09_sessions/* con excepciones.
L-2026-04-16-06 · WORKFLOW
Commit inmediato post-cambios críticos
Perder trabajo por tests background es caro. Regla: después de cada bloque funcional de cambios (fix crítico, higiene, doc sync), git commit inmediato. No esperar "al final".
