05 · Lecciones aprendidas · L-75 a L-77
Nuevas lecciones
para el LESSONS_LEARNED.md.
L-75 · Version hardcode en tests
grep antes de bump
Al hacer bump de versión, siempre correr grep recursivo por la versión vieja en todos los archivos de tests. Nunca asumir que solo __init__.py y pyproject.toml tienen el número.
L-76 · git add api/
Serverless en git o no existe
Archivos en api/ deben commitearse explícitamente. Vercel no puede desplegar lo que no está en git. Verificar git status completo antes de cada push en repos con serverless functions.
L-77 · Mintlify source of truth
.mdx en git, no en editor web
El contenido de Mintlify debe versionarse en el repo como archivos .mdx. El editor web de Mintlify es efímero. Los archivos en git son la única fuente de verdad duradera.
